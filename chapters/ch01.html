1.1 Learning Paradigms & Task Types

Supervised: trained on data with ground-truth labels (classes, depth values, key-points, next token).
Unsupervised: trained on data without labels; the model finds structure (clustering, dimensionality reduction, generative).
Classification = discrete target (assign input to one of $C$ classes; also per-pixel for semantic segmentation).
Regression = continuous target (depth estimation, facial key-points).

1.1.1Open🔥 6×Summary I.3 Q1 Summary I.3 Q2
What is the difference between (a) supervised and unsupervised learning, and (b) classification and regression?
Answer

Supervised vs unsupervised: Supervised learning trains on data with a fixed ground truth relative to the task (classes for classification, depth values for depth estimation, key-points, next letter). Unsupervised learning trains on data without any ground truth; the model finds patterns usable for clustering, dimensionality reduction or generative tasks.

Classification vs regression: Classification is a discrete goal — assign the input to one of $C$ classes (incl. semantic segmentation, one class per pixel). Regression is a continuous goal — predict a continuous value, bounded or unbounded (depth estimation, facial key-point detection).

💡 Extended memory

Memory hook: *label present → supervised; output countable → classification*. Semantic segmentation is classification done per-pixel and is supervised (needs labels) — a common trap (see SS23 1.6).

1.2 Unsupervised Methods: K-means, PCA, Autoencoders

Classic unsupervised algorithms.
K-means: iteratively assign each point to the nearest cluster centre, then recompute centres; key hyperparameter $K$.
PCA: *linear* dimensionality reduction keeping the most informative directions.
Autoencoder: a *non-linear* neural alternative for dimensionality reduction / feature learning — encoder compresses to a bottleneck $z$, decoder reconstructs; trained with reconstruction (e.g. L2) loss, no labels needed.

1.2.3Open2×Summary I.3 Q5
K-means: (a) What is the algorithm? (b) What is its key hyperparameter? (c) What are the consequences of a too small or too large value?
Answer

(a) An unsupervised algorithm clustering data into $K$ clusters.
Iterative: assign each point to the closest cluster centre, then recompute each centre; repeat until convergence.
(b) The number of clusters $K$.
(c) Too small $K$ → merges dissimilar clusters (underfitting); too large $K$ → splits similar clusters (overfitting).

💡 Extended memory

K-means alternates an assignment step and an update step (a special case of EM). It only finds a local optimum, so initialization matters; result depends on $K$ and the distance metric.

1.2.8AI-generatedOpen
You run K-means with K=1 and again with K=N (number of points). What does each degenerate case produce, and what does it tell you about choosing K?
Answer

$K=1$: every point in one cluster → the centre is the global mean; zero discrimination (extreme underfitting). $K=N$: every point is its own cluster → zero within-cluster distance but no generalization (extreme overfitting). → $K$ trades off bias and variance; use domain knowledge or an elbow/silhouette criterion.

💡 Extended memory

Within-cluster distortion monotonically decreases as $K$ grows, so you cannot pick $K$ by minimizing it — hence the elbow method.

1.3 Dataset Splits & the Validation Set

Split into train / validation / test. Train (largest, $>50\%$) is the only data the model fits. Validation (unseen) detects over/underfitting and tunes hyperparameters during training. Test is the final, kept-unbiased estimate of generalization — ideally used once. Common ratios: $\{80,10,10\}\%$, $\{70,15,15\}\%$.

A teammate suggests merging the validation set into the training set to make it bigger. What is the purpose of the validation set, and what problem will appear (and how does the validation set help avoid it)?

1.4 Shuffling, Distribution Mismatch & i.i.d.

Shuffle before splitting so every split is representative of the overall distribution. If splits come from different distributions (e.g. train = day images, test = night images) the model generalizes poorly. Mini-batches should also be i.i.d. samples of the data; a batch of consecutive/homogeneous samples gives biased gradient updates.

1.4.1Open🔥 4×SS23 7.1 SS23 7.2 SS20 P5b SS24 4.3
Your training set has only daytime images while validation/test has only nighttime images (or a batch contains only consecutive 'sunny' frames). State the problem and how to fix it. Why is shuffling before splitting important?
Answer

Problem: splits/batches come from different distributions (domain gap) → the data in a batch doesn't represent the dataset → biased gradient updates toward daytime and bad generalization / high validation error on nighttime.
Fix: mix train+val(+test), re-shuffle, and split again so all splits share the same distribution.
Why shuffle before splitting (1p): to make each split represent the general distribution and capture all the variance/cases; otherwise sorted data yields biased, mismatched splits.

*Note (SS20):* no points for “cross validation” without re-shuffling, and no points for “data augmentation” (changing brightness won't create realistic night images).

💡 Extended memory

This exact day/night scenario recurs in SS20, SS23, SS24. Key scoring words: *distribution mismatch / domain gap* and *re-shuffle then re-split*. Augmentation is explicitly the wrong fix here.

1.5 Data Leakage & Split Integrity

Any path by which test/validation information reaches training leaks and inflates your scores. Common leaks: shuffling+splitting *every epoch* (so val samples appear in train), randomly resampling train/test at test time, or computing normalization statistics over the whole dataset.

1.5.1Open2×WS24 2.1
You observe the validation loss is *smaller* than the training loss and your supervisor suspects a bug. Name a potential problem in your code or data that could cause this.
Answer

Any of:
data leakage — the validation set contains training samples; the validation set has unusually easy samples;
too-aggressive augmentation on the training set inflates the training loss; or train and validation are drawn from different distributions / the validation set is too small.

💡 Extended memory

Val < train is a red flag for leakage or an unrepresentative/easy validation split, or for heavy train-only augmentation.

1.5.2Open1×SS24 5.4

A friend reshuffles ALL data and re-splits into 80% train / 20% val *inside every epoch*. What is the problem and its consequence?

def generate_data_for_epoch(data):
    np.random.shuffle(data)            # inplace
    split = int(0.8*len(data))
    return data[:split], data[split:]  # train, val
Answer

Because the split is redrawn each epoch, samples that were in the validation set will, in later epochs, leak into the training set (and vice-versa). Over training, the model effectively sees the “validation” data, so the validation accuracy no longer reflects generalization to unseen data.

💡 Extended memory

Split once and keep the splits fixed. Re-splitting per epoch defeats the entire purpose of holding data out.

1.6 Data Augmentation

Artificially enlarge the training set by transforming inputs (flip, rotate, crop, color-jitter, add noise). It is a regularization technique that reduces overfitting and improves generalization. Applied to the training set only. It does not increase the dataset's physical size — each epoch the model sees a randomly transformed version with probability $p$. Augmentations must keep the data in the same distribution and must respect the task (don't flip MNIST vertically; don't mirror medical left/right; update the label if it is geometry-dependent).

1.6.1Open🔥 9×SS22 3.1 SS21 2.1 WS26 P2a SS20 P5c SS24 2.7 Summary I.3 Q10/Q11
Explain the concept of data augmentation: what does it do, when would you use it, and what is crucial to keep? List several augmentation techniques (e.g. for RGB images / for audio).
Answer

Concept: a regularization technique that modifies training-set inputs to introduce more variety, artificially increasing dataset size to reduce overfitting / minimize the generalization gap and improve generalization.
Use it when the dataset is too small.
Crucial: augmented data must come from the same distribution as the original data, and the transform must suit the task.
RGB-image techniques (0.5p each): rotation, cropping, flipping, color jittering / contrast change, adding Gaussian noise, translation jitter, mirroring.
Audio (SS24 2.7): pitch/EQ tuning, volume tuning, add noise / environmental sound, change speed, add effects.

💡 Extended memory

The most-examined Chapter I item — appears in nearly every exam. Always tie it to regularization and same distribution. Note it does *not* increase the physical dataset size; it shows a different random version each epoch.

1.6.2Open🔥 3×SS22 3.2 SS23 7.4 Summary I.3 Q10
To which split(s) do you apply data augmentation, and why? How does this differ from preprocessing like normalization?
Answer

Augmentation → training set only. It increases training diversity to improve generalization / reduce overfitting; validation and test must stay original and unmodified so evaluation reflects the true data distribution.
Normalization → all splits. It just rescales inputs to the range/distribution the network expects, so it must be applied consistently to train, val and test.
*(Difference, Summary Q11):* augmentation enlarges the training set; data-processing (normalization) is a fixed preprocessing applied to every split.

💡 Extended memory

One-liner: *augment train only, normalize everywhere*. Augmentation is randomized regularization; normalization is a deterministic preprocessing the model depends on at inference.

1.7 Data Processing: Normalization & Standardization

Preprocessing rescales inputs so gradients neither vanish nor explode and learning is stable.
Standardization (Z-score): $x_{std}=\dfrac{x-\mu}{\sigma}$ → zero mean, unit variance.
Min-max normalization: $x_{norm}=\dfrac{x-x_{min}}{x_{max}-x_{min}}$ → range $[0,1]$ (or $[-1,1]$). Stats ($\mu,\sigma$ or min/max) are computed on the training set only and applied to all splits.

1.7.1Open2×SS24 4.2 SS23 7.4
Before training you normalize the input images. Explain the benefit.
Answer

Normalization makes the pixel values zero-centered with unit variance (a consistent range/distribution the network expects), which makes it easier for the model to learn features, improves convergence, and prevents numerical instabilities (gradients exploding/vanishing).

💡 Extended memory

Normalized, zero-centered inputs keep activations and gradients in a healthy range, so optimization converges faster — the same motivation underlies BatchNorm later in the course.

Batch normalizationNormalization
1.7.2Open1×Summary I.2.d
Write the formulas for standardization (Z-score) and min-max normalization, and state which splits they are applied to.
Answer

Standardization: $x_{std}=\dfrac{x-\mu}{\sigma}$, with $\mu,\sigma$ the training-set mean and std → zero mean, unit variance.
Min-max normalization: $x_{norm}=\dfrac{x-x_{min}}{x_{max}-x_{min}}$ → scales to $[0,1]$.
Both are applied per sample to all splits (train/val/test), unlike augmentation; the statistics are estimated on the training set only.

💡 Extended memory

Standardization centers + scales by spread; min-max just rescales the range. Use the same train-derived constants everywhere so the model sees consistent inputs.

1.8 Datasets & Dataloaders (Programming)

A Dataset class defines how to access one sample: __len__ returns the dataset size; __getitem__(i) returns the *i*-th sample (input + label), where loading/normalization/transforms happen — it returns a single sample, not a batch. A DataLoader wraps a Dataset and handles batching, shuffling, parallel loading (workers), collation and memory pinning.

1.8.1Open2×SS23 7.5 WS26 P3e
In PyTorch, state the input and output of the Dataset's __getitem__ for supervised learning, and explain the role of the DataLoader (list 3 functionalities).
Answer

__getitem__ input (0.5p): an index/key identifying one example.
__getitem__ output (0.5p): a tuple/dict with the input instance and its label / ground truth.
DataLoader (1p): wraps the Dataset to provide, e.g., batching, shuffling, parallel/multiprocess loading (workers) — also collation and memory pinning. (The Dataset defines single-sample access; the DataLoader assembles batches.)

💡 Extended memory

__len__ + __getitem__ define the Dataset (single-sample access); the DataLoader is the batch/shuffle/parallelism engine on top. Naming batching is the key scoring point.

1.9 Class Imbalance

When some classes have far more samples than others, the model achieves high accuracy on majority classes but poor accuracy on minorities. Remedies: oversample the minority / undersample the majority, or use class weighting in the loss.

1.9.1Open2×SS22 4.10 WS26 P3d
After training, test accuracy is 95% for one class but 35% for another (or: high for label 0, poor for label 1). What is the problem and how do you address it?
Answer

Problem: an imbalanced dataset — far more samples for the easy/majority class than the minority class, so the model underperforms on the minority.
Fixes: oversample the minority class or undersample the majority; and/or use class weighting in the loss function.

💡 Extended memory

Accuracy is misleading under imbalance (a trivial majority predictor scores high). Rebalance the data or reweight the loss; report per-class metrics.
