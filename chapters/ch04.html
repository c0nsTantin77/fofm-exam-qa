4.1 Optimization Algorithms (GD, SGD, Momentum, RMSprop, Adam)

GD updates $\theta_{t+1}=\theta_t-\alpha\frac{\partial L}{\partial\theta_t}$ once per epoch (full dataset).
SGD updates every mini-batch — faster, noisier (helps escape saddle points).
Momentum accumulates a velocity to push through local minima/saddles.
RMSprop adapts a per-parameter learning rate from gradient magnitudes.
Adam = Momentum + RMSprop with bias correction. We always step in the negative gradient direction (steepest descent).

4.1.1Open🔥 4×Summary IV.4 Q1a Summary IV.4 Q1b SS20 P5e
What is the main difference between gradient descent (GD) and stochastic gradient descent (SGD)? Name two advantages of SGD.
Answer

Difference: GD performs the optimizer step at the end of each epoch (gradient over the whole training set); SGD performs a step at every iteration / mini-batch.
Advantages of SGD: (1) more optimization steps per unit time → converges faster; (2) the noisier updates help escape saddle points / local minima.

💡 Extended memory

Noise is a feature, not a bug: SGD's stochasticity is regularizing and helps escape flat/saddle regions where full-batch GD stalls.

4.1.3Open🔥 3×SS24 2.9 Summary IV.4 Q1d Summary IV.4 Q1e
What is the bias correction in Adam (and why is it needed)? Do Adam with and without bias correction have different global minima?
Answer

Adam initializes the first/second moment estimates $m,v$ at zero, so early estimates are biased toward zero, making the first updates too small/ineffective.
Bias correction $\hat m=\frac{m}{1-\beta_1^k}$, $\hat v=\frac{v}{1-\beta_2^k}$ rescales them so early steps are effective.
Global minimum: unchanged (the loss/goal is the same) — bias correction only makes early convergence faster, though the path may end in a different local minimum (SS24 2.9; Summary IV.4 Q1e: *False* they differ in global minimum).

💡 Extended memory

Bias correction matters most in the first few steps (when $\beta^k$ is far from 0). RMSprop has no bias correction simply because Adam introduced it later.

4.1.4Open2×Summary IV.4 Q1c Summary IV.4 Q1h
Why can we call RMSprop an *adaptive* optimizer? Why do we always step in the negative gradient direction, and how would you modify SGD to *maximize* the loss?
Answer

Adaptive: RMSprop adapts the learning rate per parameter, based on the running magnitude of that parameter's gradients (large gradients → smaller effective step).
Negative gradient: the gradient points in the direction of steepest ascent; to minimize we go opposite. To maximize, flip the sign: $\theta_{t+1}=\theta_t + \alpha\frac{\partial L}{\partial\theta_t}$ (gradient ascent).

💡 Extended memory

Adaptive methods (RMSprop/Adam) normalize each dimension's step, so you don't need to hand-tune a single global rate for differently-scaled parameters.

What is the major difference between Newton's method and the first-order optimizers above, and why is it not widely used for deep learning?
Answer

Newton's method is second-order — it uses the Hessian (curvature), not just the gradient. It is impractical for deep learning because computing and inverting the Hessian over millions of parameters is prohibitively expensive in compute and memory.

4.1.19Open1×WS21 5.3
How does Nesterov momentum differ from standard momentum?
Answer

Nesterov takes a look-ahead step: it first jumps in the direction of the accumulated momentum, then computes the gradient at that look-ahead position. This lets the gradient correct overshooting of the momentum within the same step, giving more responsive updates.

💡 Extended memory

Standard: gradient at current point, then add momentum. Nesterov: peek ahead via momentum, then correct.

4.1.29Open1×SS21 4.3
Explain what vanishing gradients are in deep convolutional networks and their underlying cause.
Answer

Vanishing gradients are gradients of very small magnitude, so early (shallow) layers get near-zero updates and barely learn. Causes:
saturated activations (e.g. tanh) whose derivative ≈0, and the chain rule multiplying many sub-1 factors across layers, shrinking the product toward 0.

4.2 Overfitting & Underfitting

Overfitting: great on training data, poor on unseen data (train loss ≪ val loss; val loss rises).
Underfitting: poor even on training data (both losses high/plateau). The one-word go-to fix for overfitting:
regularization. Detect via the train/val loss curves.

4.3 Regularization (L1/L2, Weight Decay, Dropout, Early Stopping)

A regularization term makes training *harder* to force generalization.
L1 pushes weights to exactly zero (sparsity);
L2 shrinks all weights toward (but not to) zero.
Weight decay is L2 applied in the optimizer step (scaled by LR).
Dropout randomly drops neurons during training (an ensemble); at inference scale by $p_{keep}$ (or use inverse dropout during training).
Data augmentation is regularization;
early stopping is debatable (it just halts, doesn't make training harder).

L1 pushes some weights to exactly zero → sparse weights.
L2 spreads the penalty so all weights become smaller, but not necessarily zero.
vs weight decay: (1) L1/L2 terms are added to the loss function, whereas weight decay is added directly to the optimizer step; (2) in weight decay the term is also scaled by the learning rate, mitigating the effect of its magnitude.

4.4 Batch Normalization

BN normalizes a layer's activations: $y=\gamma\,x_{norm}+\beta$ with learnable $\gamma,\beta$ (length = #channels/features). It stabilizes/ speeds training, reduces sensitivity to initialization, mitigates vanishing/exploding gradients, and mildly regularizes. At test time it uses running (cached) mean & variance from training since there's no batch. In a CNN (shape B×C×H×W) statistics are per-channel → $\gamma,\beta\in\mathbb{R}^{C}$.

BN keeps activations zero-mean/unit-variance per channel before the learnable rescale — the same 'keep signals in a healthy range' motivation as input normalization (Ch I).

4.5 Weight Initialization (Xavier / He)

Good init keeps activation/gradient variance stable across layers.
Xavier/Glorot suits tanh/sigmoid;
He/Kaiming $W\sim\mathcal{N}(0,\frac{2}{n_{in}})$ suits ReLU and prevents dead units. Never initialize all weights to the same value (symmetry → all neurons learn identically).

4.5.1Open🔥 3×SS21 2.4 Summary IV.4 Q6a
What weight-initialization scheme fits ReLU, and how does it affect the activations? What variance does Xavier give?
Answer

He/Kaiming initialization $W\sim\mathcal{N}\!\big(0,\frac{2}{n_{in}}\big)$ (a ReLU-tuned variant of Xavier) keeps the variance of activations stable across layers and prevents the dead-ReLU problem. Xavier keeps $\text{Var}(X_L)=\text{Var}(X_{L+1})$ so signal magnitude is preserved forward (and gradients backward).

💡 Extended memory

ReLU halves the variance (zeros out negatives), so He uses $2/n_{in}$ to compensate; Xavier ($1/n_{in}$-style) is for symmetric activations (tanh).

4.6 Hyperparameter Tuning & Learning Rate

Grid search is simple but its cost explodes with the search-space size (curse of dimensionality) — random search is often more efficient.
Learning rate is the key hyperparameter: too high → divergence; too low → painfully slow convergence. LR schedules decay the rate over training.

4.6.1Open2×SS20 P5j Summary IV.4 Q5a
Name a problem from a learning rate that is too high, and one from a rate that is too low. What is the main bottleneck of grid search?
Answer

Too high: the cost function does not converge and can even diverge.
Too low: it converges very slowly (or gets stuck), taking a very long time.
Grid search bottleneck: the search space grows exponentially with the number of hyperparameters, and each configuration needs a full (expensive) training run — impractical for large spaces.

💡 Extended memory

LR is the single most important hyperparameter. Grid search scales as (values)^(params) → use random/Bayesian search for many hyperparameters.

Learning rate
4.6.2Open2×WS24 2.12
For GPU-memory reasons you decrease your mini-batch size. Should you increase or decrease your learning rate (1p)? Why (1p)?
Answer

Decrease the learning rate. A smaller mini-batch is less representative of the full data — its gradient estimate has higher variance / more noise — so a large LR on noisy gradients risks overshooting / diverging; lowering the LR keeps updates stable.

💡 Extended memory

Smaller batch ⇒ noisier gradient ⇒ smaller, safer steps. (Linear-scaling rule: LR roughly scales with batch size.)

4.7 Transfer Learning

Reuse a model pre-trained on a large dataset as a backbone, then fine-tune on a smaller target dataset. Early layers learn generic low-level features (reusable); later layers are task-specific. Useful when labeled target data is limited.

4.7.1Open2×Summary IV.4 Q7a SS24 4.5

Answer

Autoencoder pre-training: train an autoencoder unsupervised to reconstruct a large *unlabeled* dataset, yielding an encoder that is a good feature extractor. Then remove the decoder, attach a small classifier, and train it on the smaller labeled dataset — transferring the representation learned from the big dataset.
(Also: use a pre-trained network as a backbone for a segmentation/classification task and fine-tune.)

💡 Extended memory

Two flavours: self-supervised pre-training (autoencoder) → fine-tune; or take a pretrained backbone, replace the head, freeze early layers, train on new data.
