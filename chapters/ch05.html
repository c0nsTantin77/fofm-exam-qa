5.1 CNN Architectures (LeNet, AlexNet, VGG)

LeNet — early CNN, uses average pooling & tanh/sigmoid.
AlexNet — deeper, big $11\times11$ first filter, ReLU (enabling depth) + dropout.
VGG — stacks of small $3\times3$ convs; the conv part is a feature extractor, FC layers at the end do classification. Paradigm: Conv→Pool…→Conv→Pool→FC…→FC; channels grow, spatial size shrinks with depth.

5.1.1Open2×Summary V.11 Q1a Summary V.11 Q1b
LeNet uses average pooling. Give one advantage and one disadvantage vs max pooling. What is the receptive field of a neuron in LeNet's first FC layer?
Answer

Advantage of avg pooling: every entry in the window receives a gradient.
Disadvantage: it's a linear operation, while max pooling adds non-linearity.
Receptive field of the first FC layer: the entire input image ($32\times32$) — an FC layer connects to all positions, so it sees everything.

💡 Extended memory

Any FC layer has a *global* receptive field by definition. Avg vs max pooling = linear/all-gradients vs non-linear/feature-selecting.

5.1.2Open2×Summary V.11 Q1c Summary V.11 Q1d
AlexNet uses an $11\times11$ first-layer filter and ReLU instead of sigmoid/tanh. Name two disadvantages of such a large filter, and explain why ReLU (with Kaiming init) lets AlexNet be deeper.
Answer

Large $11\times11$ filter disadvantages: (1) very expensive in parameters and computation; (2) captures global rather than fine local features.
ReLU + Kaiming: ReLU does not saturate for positive inputs, so it prevents the vanishing-gradient problem that sigmoid/tanh cause in deep nets — enabling AlexNet to train more layers than LeNet.

💡 Extended memory

Modern nets replace one big filter with stacks of small $3\times3$ ones (VGG) — same receptive field, fewer params, more non-linearity. ReLU = the depth enabler.

5.2 Skip Connections & ResNet

A residual block computes $X_{l+1}=X_l+F(X_l)$. The identity path is a 'highway' for gradients: $\frac{\partial L}{\partial X_l}=\frac{\partial L}{\partial X_{l+1}}\big(1+\frac{\partial F(X_l)}{\partial X_l}\big)$ — the $+1$ lets the gradient bypass the block, mitigating vanishing gradients and enabling very deep nets.

5.2.1Open🔥 3×Summary V.11 Q2a Summary V.11 Q2b
Why do skip connections create a 'highway of gradients'? Show it in the chain rule for $\frac{\partial L}{\partial X_l}$ given $X_{l+1}=X_l+F(X_l)$.
Answer

Skip connections add the input to the output, so the gradient of that input can bypass the block instead of being diminished by it:
$\frac{\partial L}{\partial X_l}=\frac{\partial L}{\partial X_{l+1}}\frac{\partial X_{l+1}}{\partial X_l}=\frac{\partial L}{\partial X_{l+1}}\Big(\frac{\partial X_l}{\partial X_l}+\frac{\partial F(X_l)}{\partial X_l}\Big)=\frac{\partial L}{\partial X_{l+1}}\Big(1+\frac{\partial F(X_l)}{\partial X_l}\Big).$
The $1$ guarantees gradient flow even if $\frac{\partial F}{\partial X_l}\to0$.

💡 Extended memory

That additive $+1$ is the whole trick: the gradient is never fully attenuated, so 100+ layer ResNets train. Same idea reappears in LSTM cell state.

5.2.3Open2×SS25 6.5
For a residual block $H(x)=x+F(x)$ with $F=F_2(F_1(x))$, compute $\frac{\partial L}{\partial x}$ and explain what happens when $\frac{\partial F}{\partial x}\to0$.
Answer

$\frac{\partial L}{\partial x}=\frac{\partial L}{\partial H}\big(1+\frac{\partial F}{\partial x}\big)$. When the inner gradients vanish ($\frac{\partial F}{\partial x}\to0$), $\frac{\partial L}{\partial x}\to\frac{\partial L}{\partial H}$ — the gradient passes straight through the block (the skip path), so it doesn't vanish.

💡 Extended memory

The additive identity gives a guaranteed gradient highway: even if the block learns nothing, gradients still flow.

5.3 GoogLeNet / Inception

Inception runs multiple kernel sizes in parallel and concatenates them. The first version was too expensive (large kernels);
1×1 convolutions were added to reduce channels before costly $3\times3/5\times5$ convs. MaxPool inside the block uses the size-preserving trio $(k{=}3,s{=}1,p{=}1)$ so its output can be concatenated.

5.3.1Open2×Summary V.11 Q1f
Inception: (i) what was the problem with the first version and how was it solved? (ii) MaxPool usually reduces spatial size — how was it used *inside* an Inception block and concatenated to the other outputs?
Answer

(i) The first InceptionNet was too complex/expensive (large-kernel convolutions). Solved by inserting 1×1 convolutions to reduce the channel count before the more expensive convs (a bottleneck).
(ii) By using the size-preserving trio $k{=}3,s{=}1,p{=}1$, the MaxPool keeps the spatial dimensions, so its output matches the others and can be concatenated.

💡 Extended memory

1×1 bottlenecks made 'going deeper' (GoogLeNet) affordable. The size-preserving trio is reused everywhere to keep parallel branches concatenable.

5.4 Autoencoders

An autoencoder encodes input to a bottleneck $z$ then decodes back, trained with a reconstruction loss (MSE/MAE) — unsupervised (input is its own target). Latent too small → blurry/underfit; too big → learns identity/overfits. It is the non-linear analogue of PCA; great for unsupervised feature extraction on unlabeled data.

5.4.1Open🔥 3×Summary V.11 Q3a Summary V.11 Q3c SS22 5.3
For an image-reconstruction autoencoder: what loss can you use, what do you compare, and is it supervised or unsupervised? What linear method does it resemble and what's its advantage?
Answer

Loss: MSE or MAE, comparing the input image with the reconstructed output. It is unsupervised (no labels — the input is the target).
Resembles PCA (linear dimensionality reduction); the autoencoder's advantage is that it is non-linear and can learn more complex features.

💡 Extended memory

Autoencoder = non-linear PCA. 'No ground-truth labels needed' is the recurring exam phrase (also Ch I, SS22 5.3).

5.5 U-Net & Fully Convolutional Networks

U-Net is an encoder–decoder with skip connections between matching encoder/decoder levels, preserving fine spatial detail lost during downsampling. It accepts variable input sizes (fully convolutional) and processes most computation at low resolution. Channel count is increased to compensate for reduced spatial size.

5.5.1Open2×Summary V.11 Q4a Summary V.11 Q4b
Give advantages of U-Net over a vanilla autoencoder. How do we mitigate the drop in spatial size so not too much information is lost?
Answer

Advantages: (i) skip connections between encoder and decoder preserve fine detail; (ii) most processing happens at lower resolution (cheaper); (iii) accepts different input sizes with no structural change; (iv) extracts meaningful local features instead of only global ones.
Mitigation: increase the number of channels as spatial size shrinks, giving features more 'breathing room' to pass through.

💡 Extended memory

Skip connections in U-Net carry high-resolution detail across the bottleneck — crucial for segmentation where pixel-precise boundaries matter.

5.5.2Open1×Summary V.11 Q4c
For plain image *reconstruction*, does it make sense to use encoder–decoder skip connections? Explain.
Answer

No — for pure reconstruction, skip connections let the model copy the input straight to the output (memorize/identity), so it never learns a meaningful compressed representation. Skip connections help segmentation (need detail) but defeat the purpose of a compressing autoencoder.

💡 Extended memory

Skip connections + reconstruction target = trivial identity shortcut. They're beneficial only when the output needs the bypassed high-res detail (segmentation).

5.6 Variational Autoencoders (VAEs)

VAEs learn the data distribution explicitly. Two losses: a reconstruction loss (MSE/MAE) + a KL-divergence term forcing the latent toward a standard normal. The reparameterization trick $z=\mu+\sigma\epsilon$ (with $\epsilon\sim\mathcal{N}(0,1)$) makes sampling differentiable so backprop works.

5.6.1Open2×Summary V.11 Q5d Summary V.11 Q5e Summary V.11 Q5f
VAE: what two losses are used and their purpose? What distribution do we assume for the latent? How does the reparameterization trick enable sampling?
Answer

Two losses: (1) reconstruction (MAE/MSE) to match the output to the input; (2) KL divergence to push the latent distribution toward the assumed prior.
Assumed distribution: the standard normal $\mathcal{N}(0,1)$.
Reparameterization trick: sample $z=\mu+\sigma\epsilon$ with $\epsilon\sim\mathcal{N}(0,1)$ — the randomness is moved into $\epsilon$, so $\mu,\sigma$ stay differentiable and gradients can flow (you can't backprop through a raw random sample).

💡 Extended memory

KL keeps the latent space smooth/continuous so you can sample & interpolate; the reparam trick is what makes the stochastic node trainable by backprop.

What are the main differences between a standard autoencoder and a variational autoencoder?
Answer

A standard AE imposes no structure on the latent space — it only compresses and reconstructs the input (reconstruction loss only). A VAE imposes a prior on the latent space (learns a probability distribution), samples from the latent distribution, and adds a KL-divergence term to the loss → enabling generation of new samples.

💡 Extended memory

VAE = AE + a regularized, samplable latent (KL to a prior). Plain AE's latent has 'holes', so it can't generate.

What is the difference between how GANs and VAEs learn the training-set distribution?
Answer

VAEs learn the distribution explicitly (directly) — they model $p(x)$ via an encoder/decoder and a KL-regularized latent.
GANs learn it implicitly (indirectly) — the generator learns to produce samples that fool a discriminator, never writing down $p(x)$.

Explicit (VAE) → smoother, blurrier samples, tractable latent. Implicit (GAN) → sharper samples, harder/unstable training.

5.7 Generative Adversarial Networks (GANs)

A GAN pits a generator (turns noise into fake images) against a discriminator (real vs fake), trained min-max. Discriminator loss is BCE on real(1)/fake(0); to train the generator we relabel fakes as 1 to fool D. Prone to underfitting and mode collapse (generator outputs one image that fools D). GANs give sharper images than VAEs but are unstable to train.

5.7.1Open2×Summary V.11 Q5b Summary V.11 Q5c
In the vanilla GAN, why is it prone to underfitting and mode collapse? How do we train the generator to fool the discriminator?
Answer

Underfitting / mode collapse: generator and discriminator work against each other in a min-max game, so both can converge to suboptimal solutions; the generator may learn to produce a single image that reliably fools the discriminator (mode collapse), ignoring diversity.
Training the generator: pass the fake image through the discriminator but label it as class 1 (real), so the generator is pushed to make fakes the discriminator scores as real.

💡 Extended memory

The 'relabel fakes as real' is the generator's objective. Mode collapse = generator exploits one D-fooling sample; WGAN/conditional GAN variants address instability.

GAN
5.7.3Open1×Summary V.10
Write the discriminator and generator losses of a vanilla GAN.
Answer

Discriminator (BCE, real=1/fake=0): $L_D=-\frac{1}{2N}\sum_i\big(y_i\log D(x_i)+(1-y_i)\log(1-D(G(z_i)))\big)$.
Generator (relabel fakes as real): $L_G=-\frac{1}{N}\sum_i\log D(G(z_i))$ — maximize the discriminator's probability that the fake is real.

💡 Extended memory

D wants $D(\text{real})\to1, D(\text{fake})\to0$; G wants $D(\text{fake})\to1$. The two losses pull D's output on fakes in opposite directions.

During GAN training the discriminator becomes near-perfect early (D≈1 on real, ≈0 on fake). Why is this a problem for the generator?
Answer

A near-perfect discriminator gives the generator vanishing gradients (its loss saturates), so the generator stops learning. Remedies: balance D/G training, use a non-saturating or Wasserstein loss, or weaken D.
