Recurrent Neural Networks (RNNs)

A vanilla RNN cell is $h_t=\tanh(V h_{t-1}+W x_t+b)$ — the same weights are reused at every step. Parameters in a cell with $x_t\in\mathbb{R}^{d}$, $h_t\in\mathbb{R}^{m}$: $d\!\cdot\!m + m\!\cdot\!m + m$ (input, recurrent, bias). RNNs suffer vanishing/exploding gradients over long sequences, have few parameters (limited capacity), and short-term memory.

Open2×SS20 P6a
For a vanilla RNN $h_t=\tanh(V h_{t-1}+W x_t+b)$ with $x_t\in\mathbb{R}^3$ and $h_t\in\mathbb{R}^5$, how many parameters are in the cell?
Answer

$W$ ($3\times5$) + $V$ ($5\times5$) + bias ($5$) $= 15 + 25 + 5 = \mathbf{45}$.

💡 Extended memory

Count the three pieces: input-to-hidden ($d\cdot m$), hidden-to-hidden ($m\cdot m$), and bias ($m$). The recurrent matrix $V$ is square ($m\times m$).

Open2×Summary VI.5 Q1
Give two drawbacks of RNNs that are NOT the exploding/vanishing gradient problem.
Answer

(1) Too few parameters — limited capacity (the same small weight set is reused at every timestep). (2) Short-term memory — they struggle to learn relationships between tokens that are far apart in the sequence.

💡 Extended memory

Beyond gradient issues, RNNs forget long-range context (short memory) and have limited capacity — both motivate LSTMs and then attention.

Open2×SS22 9.1
RNNs over long sequences exhibit a training issue. What is the problem, and how do you address the vanishing vs exploding cases?
Answer

The problem is vanishing / exploding gradients (gradients shrink or blow up as they backprop through many timesteps).
If vanishing: switch to an LSTM (cell-state gradient highway), or enforce the recurrent matrix eigenvalues ≈ 1.
If exploding: apply gradient clipping.

💡 Extended memory

Backprop-through-time multiplies many Jacobians → product shrinks (vanish) or grows (explode). Clipping caps explosion; LSTM/gating fixes vanishing.

Open2×SS25 7.1 SS25 7.2
Write the (activation-free) RNN cell-state update $C(t)$, then unroll it to show $C(t)$ depends on all previous inputs.
Answer

$C(t)=W_c\,C(t{-}1)+W_x X_t$. Unrolling: $C(t)=\sum_{i=0}^{t} W_c^{\,t-i} W_x X_i = W_c^t W_x X_0 + \dots + W_x X_t$ — every past input $X_i$ contributes, weighted by $W_c^{\,t-i}$.

💡 Extended memory

The $W_c^{\,t-i}$ factor is exactly why gradients vanish ($\|W_c\|<1$) or explode ($>1$) over long sequences.

Open1×SS20 P6b
For $h_t=\tanh(V h_{t-1}+W x_t+b)$: if $x_t$ and $b$ are the zero vector, is it true that $h_t=h_{t-1}$ for any $h_t$?
Answer

False. Even with $x_t=0$ and $b=0$, $h_t=\tanh(V h_{t-1})$ — the multiplication by $V$ and the non-linearity $\tanh$ change the value, so in general $h_t\neq h_{t-1}$.

💡 Extended memory

Only if $V$ were identity *and* there were no non-linearity would the state pass through unchanged. The recurrent weight + tanh transform the hidden state every step.

Open1×SS25 7.3
An RNN does well on short sentences but worse on long ones. (a) Name the phenomenon. (b) Name a major cause.
Answer

(a) Loss of long-term memory / vanishing (or exploding) gradients. (b) Old input embeddings are multiplied by the weight matrix repeatedly (and passed through the activation many times), so their influence shrinks (or blows up) over many steps.

💡 Extended memory

Repeated multiplication by $W_c$ across timesteps = exponential decay/growth → short memory. LSTMs add a cell-state highway to fix it.

Why does using the *same* weights at every timestep make RNNs prone to vanishing/exploding gradients?
Answer

Backpropagation through time repeatedly multiplies by the same recurrent Jacobian (involving $V$). Over $T$ steps this is like raising $V$ to the $T$-th power — eigenvalues $<1$ shrink the gradient toward 0 (vanishing) and eigenvalues $>1$ blow it up (exploding), exponentially in sequence length.

💡 Extended memory

Weight sharing across time = repeated multiplication = exponential growth/decay. Eigenvalues ≈ 1 is the knife-edge that keeps gradients stable.

LSTM

An LSTM adds a cell state $C_t$ — a gradient highway analogous to skip connections — plus gates (forget, input, output) using sigmoid to decide what to keep/add/output. This lets it carry long-range information and largely solves the vanishing-gradient problem of vanilla RNNs.

Open2×Summary VI.5 Q2
How did the LSTM solve the vanishing-gradient problem of RNNs?
Answer

By introducing the cell state — a highway for gradients (just like skip connections in ResNets). Information (and gradients) can flow along the cell state across many timesteps with minimal attenuation, so long-range dependencies are preserved.

💡 Extended memory

Same core trick as ResNet's $+1$: an additive, near-identity path for gradients. The gates control what enters/leaves this highway.

Open2×Summary VI.5 Q3
Is it a good idea to use ReLU instead of Sigmoid as the activation of the LSTM input gate?
Answer

No. The sigmoid is bounded in $[0,1]$, so it acts as a gate deciding *what fraction* of information to pass (0 = block, 1 = pass). ReLU is unbounded above, which destroys this gating behaviour and can cause numerical instabilities.

💡 Extended memory

Gates need a $[0,1]$ 'valve' → sigmoid. Candidate values (what to write) use tanh. ReLU's unbounded range breaks the valve metaphor.

Open1×SS25 2.6
Explain the role of the sigmoid activation in an LSTM unit.
Answer

The sigmoid acts as a gate: in the input/forget/output gates it outputs values in $[0,1]$ — like a soft switch — deciding how much information to keep or forget.

💡 Extended memory

Sigmoid ∈ [0,1] = a valve; that's why ReLU (unbounded) is a poor gate activation.

Name the three gates of an LSTM and what each controls.
Answer

Forget gate — how much of the previous cell state $C_{t-1}$ to keep. Input gate — how much of the new candidate information to write into the cell state. Output gate — how much of the cell state to expose as the hidden state $h_t$. All three use sigmoids to produce $[0,1]$ gating values.

💡 Extended memory

Mnemonic: forget (erase old), input (write new), output (read out). The cell state is updated as $C_t = f\odot C_{t-1} + i\odot \tilde C_t$.

Transformers: Attention & Architecture

Transformers replace recurrence with attention. Self-attention computes $\text{softmax}\!\big(\frac{QK^\top}{\sqrt{d_k}}\big)V$. The $\sqrt{d_k}$ scaling and layer normalization (before/after attention blocks) keep gradients stable. Cross-attention in the decoder uses queries from the decoder and keys/values from the encoder. Predicting the next token is a classification task (one label per vocabulary word).

Open2×Summary VI.5 Q4e
Why is the self-attention mechanism prone to exploding gradients, and how does the original Transformer architecture solve it?
Answer

Self-attention itself has no normalization besides the $\frac{1}{\sqrt{d_k}}$ scale, so the affine layers can produce very large values → huge gradients. The Transformer adds layer normalization (around the attention and feed-forward blocks), normalizing each embedding individually to keep activations/gradients stable.

💡 Extended memory

Two stabilizers: the $\sqrt{d_k}$ scale on the scores, and LayerNorm + residual connections around each sub-block. LayerNorm (per-token) suits variable-length sequences better than BatchNorm.

Open2×Summary VI.5 Q4c Summary VI.5 Q4d
Cross-attention: given encoder outputs $X_e\in\mathbb{R}^{N\times M}$ and decoder outputs $X_d\in\mathbb{R}^{K\times M}$, what is the output dimension? Is predicting the next token classification or regression?
Answer

Use $q=X_d$, $k=X_e$, $v=X_e$: $\text{out}=(q\,k^\top)\,v \Rightarrow (K\times M)(M\times N)(N\times M)=(K\times N)(N\times M)=\mathbf{K\times M}$.
Next-token prediction is classification — each word in the vocabulary is its own class.

💡 Extended memory

Cross-attention lets each decoder position ($K$) attend over all encoder positions ($N$); output keeps the decoder length $K$. Language modeling = classification over the vocabulary.

Open2×SS25 2.4 SS25 7.4
Why are Transformers better suited than RNNs for GPU parallelism, and why do they capture long-range dependencies well?
Answer

Parallelism: self-attention computes interactions between all tokens simultaneously, so the sequence is processed in parallel, whereas RNNs are sequential (each step waits for the previous one). Long-range: every token can directly attend to every other token, regardless of distance, so far-apart dependencies are captured in one step (no multi-step decay).

💡 Extended memory

Attention = parallel + direct token-to-token links; RNN = sequential + distance-decayed memory.

Open1×Summary VI.5 Q4a Summary VI.5 Q4b
Can a Transformer take embeddings of different sizes? Can it take sequences of different lengths at the encoder/decoder?
Answer

Embedding size: No — the internal fully-connected layers expect a fixed embedding dimension.
Sequence length: Yes — sequences of embeddings are like batches of tokens, and the FC layers handle different batch/sequence sizes; cross-attention also handles differing encoder/decoder lengths.

💡 Extended memory

Fixed feature width, flexible sequence length — the opposite of FC-based CNNs (fixed spatial size). Attention is permutation/length-agnostic by construction.

Open1×SS25 7.5
In $\text{Attention}(Q,K,V)=\text{softmax}\!\big(\frac{QK^\top}{\sqrt d}\big)V$ with $Q,K,V\in\mathbb{R}^{n\times d}$, is softmax applied over rows or columns? Why?
Answer

Over the rows of the attention matrix. Each query's output is a weighted sum of values, and those weights must sum to 1 — softmax over the row (across all keys) normalizes the weights for that query.

💡 Extended memory

Row-wise softmax ⇒ each query distributes attention (summing to 1) over all keys.

Transformers stack many attention layers and can suffer vanishing gradients. (a) What fix (borrowed from deep CNNs) helps? (b) Is the RNN vanishing-gradient cause the same?

Why is the dot-product attention score scaled by $\frac{1}{\sqrt{d_k}}$ before the softmax?
Answer

For large key dimension $d_k$, dot products grow large in magnitude, pushing softmax into saturated regions with tiny gradients. Dividing by $\sqrt{d_k}$ keeps the scores at a reasonable scale, so softmax gradients stay healthy.

💡 Extended memory

The $\sqrt{d_k}$ scale stabilizes attention; combined with LayerNorm it prevents exploding gradients.

Positional Encoding

Self-attention is permutation-invariant — the scores don't depend on token order. To inject order, Transformers add positional encodings to the input embeddings, unlike RNNs which process tokens sequentially and get order for free.

Open2×Summary VI.5 Q4f
Why is positional encoding important in Transformers, in comparison to RNNs?
Answer

Transformers do not process the sequence sequentially — the attention scores between two embeddings are invariant to token order (permutation-invariant). So we must add positional encoding to inject time/order information. RNNs get this for free because they consume tokens one-by-one in order.

💡 Extended memory

No recurrence ⇒ no inherent order ⇒ add positional encodings. Without them, 'dog bites man' and 'man bites dog' would look identical to attention.

Open1×Summary III.5 Q13
To feed a CNN feature map ($B\times C\times H\times W$) into a Transformer, how do you turn pixels into tokens?
Answer

Move channels last and flatten the spatial dims: $(B\times C\times H\times W)\to(B\times H\times W\times C)\to(B\times(H\cdot W)\times C)$. Each pixel becomes a $C$-dimensional token, giving a sequence of $H\cdot W$ tokens per image.

💡 Extended memory

This 'flatten spatial → tokens' reshape is the bridge from CNNs to Vision Transformers (cross-listed from Ch III).

Open1×WS23 2.2
Why is positional encoding used in Transformers, and what problem arises without it?
Answer

It adds information about token position in the sequence. Without it, attention treats the input as an unordered set (no inherent notion of order), so the model can't use word order — yet order changes meaning ('dog bites man' ≠ 'man bites dog').

💡 Extended memory

No recurrence/convolution ⇒ no built-in order ⇒ positional encoding supplies it.
