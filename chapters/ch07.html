7.1 Gradients, Jacobians & the Scalar-Loss Principle

For $f:\mathbb{R}^{n\times m}\to\mathbb{R}$ (scalar output), $\frac{\partial f}{\partial x}$ is a gradient with the same shape as $x$. For $f:\mathbb{R}^n\to\mathbb{R}^m$ (vector output) it is a Jacobian ($m\times n$). Because a neural network always ends in a scalar loss $L\in\mathbb{R}$, we only ever compute gradients (same shape as the variable) — never full Jacobians.

7.1.1Open2×Summary VII.1 Summary II.8 Q4a
Why can backpropagation avoid computing full Jacobians, and what shape does the gradient of the loss w.r.t. a weight matrix have?
Answer

Because the loss is a scalar $L\in\mathbb{R}$. The derivative of a scalar by a matrix is a gradient with the same shape as that matrix (each entry $\frac{\partial L}{\partial w_{ij}}$ is a scalar), not a Jacobian. So $\frac{\partial L}{\partial W}$ has the same shape as $W$.
A Jacobian would only appear for a vector→vector map (e.g. $\frac{\partial Y}{\partial W}$ alone), but the scalar loss collapses it.

💡 Extended memory

Shape-matching rule: $\frac{\partial L}{\partial v}$ has the same shape as $v$, for any variable $v$. This is the single most useful sanity check in backprop.

7.1.2Open1×Summary VII.1
Distinguish a gradient from a Jacobian. Give the shapes for $f:\mathbb{R}^n\to\mathbb{R}$ and $f:\mathbb{R}^n\to\mathbb{R}^m$.
Answer

Gradient (scalar output $f:\mathbb{R}^n\to\mathbb{R}$): a vector $\frac{\partial f}{\partial x}\in\mathbb{R}^n$ (same shape as input).
Jacobian (vector output $f:\mathbb{R}^n\to\mathbb{R}^m$): a matrix $\frac{\partial f}{\partial x}\in\mathbb{R}^{m\times n}$ (one row per output, one column per input).

💡 Extended memory

Scalar-out → gradient (input-shaped). Vector-out → Jacobian (out×in). Deep learning lives in the gradient world thanks to the scalar loss.

Jacobian

7.1.4AI-generatedOpen
For a layer $y=f(x)$ with $x\in\mathbb{R}^n,\ y\in\mathbb{R}^m$, what is the shape of $\frac{\partial y}{\partial x}$? After composing with a scalar loss $L$, what is the shape of $\frac{\partial L}{\partial x}$?
Answer

$\frac{\partial y}{\partial x}$ is the Jacobian, shape $m\times n$. After the scalar loss, $\frac{\partial L}{\partial x}$ is a gradient of shape $n$ (same as $x$): $\frac{\partial L}{\partial x}=\big(\frac{\partial y}{\partial x}\big)^\top\frac{\partial L}{\partial y}$.

💡 Extended memory

Vector→vector gives a Jacobian; the scalar loss contracts it into an input-shaped gradient.

7.2 Affine-Layer Derivatives

For $Y=XW+b$ with upstream gradient $\text{dout}=\frac{\partial L}{\partial Y}$ (shape $N\times M$): $\frac{\partial L}{\partial W}=X^\top\,\text{dout}$ ($D\times M$), $\frac{\partial L}{\partial X}=\text{dout}\,W^\top$ ($N\times D$), $\frac{\partial L}{\partial b}=\mathbf{1}_N^\top\,\text{dout}=\sum_{n}\text{dout}$ ($1\times M$).

7.2.1Open🔥 3×Summary VII.1 SS22 2.2 SS22 2.3
For $Y=XW+b$ ($X_{N\times D}$, $W_{D\times M}$, $b_{1\times M}$) and a scalar loss $L$, derive $\frac{\partial L}{\partial W}$ and $\frac{\partial L}{\partial X}$ with their shapes.
Answer

$\frac{\partial L}{\partial W}=X^\top\cdot\frac{\partial L}{\partial Y}=X^\top\,\text{dout}$, shape $D\times M$ (same as $W$).
$\frac{\partial L}{\partial X}=\frac{\partial L}{\partial Y}\cdot W^\top=\text{dout}\,W^\top$, shape $N\times D$ (same as $X$).
Both hold because $L$ is a scalar — otherwise $\frac{\partial Y}{\partial W}$ would be a Jacobian tensor.

💡 Extended memory

Memorize by shape: to get $D\times M$ from $X(N\times D)$ and $\text{dout}(N\times M)$ you must use $X^\top\text{dout}$. The transpose and order are forced by shape-matching.

BackpropagationJacobian
7.2.2Open2×Summary VII.1
How is the bias $b$ handled in $Y=XW+b$ (it's $1\times M$ but $XW$ is $N\times M$), and what is $\frac{\partial L}{\partial b}$?
Answer

$b$ is broadcast: mathematically $Y=XW+\mathbf{1}_N b$, where $\mathbf{1}_N$ is a column of ones that copies $b$ across all $N$ rows (this is exactly NumPy broadcasting).
Treating $\mathbf{1}_N b$ like another $XW$, $\frac{\partial L}{\partial b}=\mathbf{1}_N^\top\cdot\frac{\partial L}{\partial Y}=\sum_{i=1}^{N}\text{dout}_{i,:}$ — i.e.
sum the upstream gradient over the batch → shape $1\times M$.

💡 Extended memory

Bias gradient = column-sum of dout (np.sum(dout, axis=0)), because $b$ contributed identically to every row of the batch.

Backpropagation
7.2.3Open2×SS25 4.3
Given upstream $\frac{\partial L}{\partial z_1}$ for $z_1=W_1 x_i+b_1$:
(a) write $\frac{\partial L}{\partial W_1},\frac{\partial L}{\partial x_i},\frac{\partial L}{\partial b_1}$;
(b) which need not be computed in training, and why?
Answer

(a) $\frac{\partial L}{\partial W_1}=\frac{\partial L}{\partial z_1}x_i^\top$, $\frac{\partial L}{\partial x_i}=W_1^\top\frac{\partial L}{\partial z_1}$, $\frac{\partial L}{\partial b_1}=\frac{\partial L}{\partial z_1}$.
(b) $\frac{\partial L}{\partial x_i}$ is not needed — it's the gradient w.r.t. the input, which is not a learnable parameter (you'd only need it to pass gradients further back, but here $x_i$ is the network input).

💡 Extended memory

You only need gradients w.r.t. parameters to update them; the input gradient matters only for propagating to earlier layers.

Backpropagation

Note the shapes: $\partial L/\partial W$ matches $W$, $\partial L/\partial X$ matches $X$; the bias gradient sums the upstream over the batch.

2-layer MLP (single sample): $a=W_1x+b_1,\ h=\text{ReLU}(a),\ z=W_2h+b_2,\ p=\text{softmax}(z)$, loss $L=-\ln p_y$ with $y=2$. Given $p=[\tfrac14,\tfrac12,\tfrac14]^\top,\ a=[\tfrac12,-1]^\top,\ \frac{\partial L}{\partial h}=[\tfrac12,-\tfrac14]^\top$:
(a) loss $L$;
(b) $\frac{\partial L}{\partial z}$;
(c) $\frac{\partial L}{\partial a}$ and which hidden unit gets zero gradient and why.
Answer

(a) $L=-\ln p_2=-\ln\tfrac12=\ln 2$.
(b) softmax+CE gives $\frac{\partial L}{\partial z}=p-\mathbb{1}_y=[\tfrac14,\tfrac12,\tfrac14]-[0,1,0]=[\tfrac14,-\tfrac12,\tfrac14]^\top$.
(c) $\frac{\partial L}{\partial a}=\frac{\partial L}{\partial h}\odot\mathbb{1}[a>0]=[\tfrac12,-\tfrac14]\odot[1,0]=[\tfrac12,0]^\top$;
hidden unit 2 gets zero gradient because its pre-activation $a_2=-1<0$ was deactivated by ReLU (derivative 0 there).

💡 Extended memory

softmax+CE backward is $p-y$; ReLU backward masks gradients where the pre-activation was negative.

7.2.7Open1×Summary VII.1
Why is the per-weight derivative $\frac{\partial L}{\partial w_{u,v}}=\sum_i\sum_j\frac{\partial L}{\partial y_{i,j}}\frac{\partial y_{i,j}}{\partial w_{u,v}}$ a double sum?
Answer

Because $w_{u,v}$ can influence multiple entries $y_{i,j}$ of the output, and the chain rule must accumulate the contribution through every output entry it affects. The scalar $\frac{\partial L}{\partial w_{u,v}}$ sums all those paths — this is the 'each neuron collects/sums the upstream gradients it took part in' picture.

💡 Extended memory

Each parameter's gradient = sum over all outputs it touched. For $Y=XW$, $w_{u,v}$ only affects column $v$, which is why the matrix form collapses to $X^\top\text{dout}$.

Backpropagation

Given $X\in\mathbb{R}^{4\times3}$, $W\in\mathbb{R}^{3\times5}$ and $\text{dout}\in\mathbb{R}^{4\times5}$, state the shapes of $\frac{\partial L}{\partial W}$, $\frac{\partial L}{\partial X}$, $\frac{\partial L}{\partial b}$ and how to compute each.
Answer

Each gradient matches its variable's shape (W:3×5, X:4×3, b:1×5). Use that to decide transposes/multiplication order without re-deriving.

7.3 Chain Rule & the Upstream Gradient (dout)

Backprop treats each layer in its own scope. The upstream gradient $\text{dout}=\frac{\partial L}{\partial Y}$ (already computed by the layers above) enters the layer; the layer computes its parameter gradients and passes a new upstream gradient $\frac{\partial L}{\partial X}$ to the layer below.

7.3.1Open2×Summary VII.1
Explain the 'upstream gradient' (dout) abstraction in backpropagation for a layer like Affine → ReLU → Affine → Sigmoid → Loss.
Answer

When deriving a given layer (say the Affine layer), we treat it as a function with its own scope. Everything above it has already produced $\text{dout}=\frac{\partial L}{\partial Y}$ (the upstream gradient). The layer uses dout to compute its local parameter gradients ($\frac{\partial L}{\partial W}, \frac{\partial L}{\partial b}$) and the gradient to send further down ($\frac{\partial L}{\partial X}$), which becomes the next layer's dout.

💡 Extended memory

Modular backprop: each layer only needs its forward inputs + the incoming dout. This local abstraction is exactly how autograd frameworks implement backward().

7.3.2Open1×Summary VII.1
Compute a tiny chain-rule example: $f(X)=2X$, $g(z)=\sum_i z_i$, $L=g(f(X))$ with $X\in\mathbb{R}^n$. What is $\frac{\partial L}{\partial x_1}$?
Answer

$L=2x_1+2x_2+\dots+2x_n$. Then $\frac{\partial L}{\partial x_1}=\sum_k\frac{\partial L}{\partial f(x)_k}\frac{\partial f(x)_k}{\partial x_1}=2\cdot1+2\cdot0+\dots+2\cdot0=\mathbf{2}$ (only $f(x)_1=2x_1$ depends on $x_1$).

💡 Extended memory

The chain rule sums over every intermediate that depends on $x_1$; here only one term survives because $f$ is element-wise.

Backpropagation
7.3.3Open1×SS25 4.4
At one iteration $z_2\approx10^6$ for all samples before a sigmoid output. Estimate the gradient of the learnable parameters.
Answer

≈ 0. The sigmoid is deep in its saturated 'dead zone' where $\sigma'(z_2)\approx0$, so every upstream gradient gets multiplied by ~0 → the parameter gradients are essentially zero (no learning this step).

💡 Extended memory

Saturated sigmoid/tanh kills gradients (vanishing). Keep pre-activations near 0 via normalization/good init.

7.3.4Open1×WS21 7.4
Two identical trainings differ only in that one normalizes the loss by $1/N$. Does this change the optimal weights? After 10 SGD steps, are the weights the same? Explain.
Answer

The optimum is unchanged — scaling the loss by a constant doesn't move its minimizer. But after 10 steps the weights differ, because scaling the loss scales every gradient by $1/N$, which is equivalent to using a different (smaller) effective learning rate — so the two trajectories diverge.

💡 Extended memory

Constant loss scaling ≡ rescaling the LR: same minimum, different path/speed under gradient descent.

7.3.5Multiple Choice1×WS23 1.9
Which statements about backpropagation are true?

Select the options you think are correct (it may be none), then check.

✅ Correct: A, B
Answer

True: the gradient through tanh is always ≤ the upstream gradient in magnitude ($\tanh'\le1$), and sigmoid/tanh/Leaky-ReLU don't flip the gradient's sign (their derivatives are positive).
False: vanishing gradients make earlier layers learn slower (not deeper); and for $\partial L/\partial w=-6$, *decreasing* $w$ *increases* the loss.

💡 Extended memory

tanh shrinks gradients (≤1), positive-derivative activations preserve sign; vanishing hits shallow layers.

7.3.6AI-generatedOpen
In a Conv→ReLU→Conv block, the second conv receives dout. List what it must compute and what it passes to the ReLU below it.
Answer

The second conv uses its incoming $\text{dout}=\frac{\partial L}{\partial Y}$ to compute its parameter gradients ($\frac{\partial L}{\partial W},\frac{\partial L}{\partial b}$) and its input gradient $\frac{\partial L}{\partial X}$; it then passes $\frac{\partial L}{\partial X}$ down as the dout for the ReLU below.

💡 Extended memory

Each layer: receive dout → compute local param grads → emit input-grad as the next layer's dout. This locality is exactly how autograd works.
